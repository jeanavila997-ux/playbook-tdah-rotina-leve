# Mapa do repositório

```text
playbook-tdah-rotina-leve/
├── .github/              # validação e colaboração no GitHub
├── assets/               # identidade visual e orientação de imagens
├── automacoes/           # workflows de captura e venda
├── backend/              # API opcional de leads
├── criativos/            # templates por canal
├── dados/                # modelos vazios de métricas
├── docs/                 # produto, público, operação e conformidade
├── ebook/                # material gratuito
├── funil/                # jornada, e-mails e rastreamento
├── landing-page/         # copy e mapa da captura
├── pagina-vendas/        # oferta e copy comercial
├── playbook/             # produto principal
├── scripts/              # validação e empacotamento
├── site/                 # site pronto para hospedagem
└── templates/            # modelos reutilizáveis
```

## Regra de separação

- O que publica na web fica em `site/`.
- O conteúdo-fonte fica em `ebook/` e `playbook/`.
- Estratégia comercial fica em `funil/`, `criativos/`, `landing-page/` e `pagina-vendas/`.
- Modelos ficam em `templates/`.
- Decisões e orientações internas ficam em `docs/`.
- Dados reais, credenciais e backups privados nunca entram no GitHub.
